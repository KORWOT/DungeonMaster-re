기획서가 최우선이로 뭔가 의문점이나 더 필요한 내용이 있으면 제시하고 자세한 내용은 아래 참고해줘



📄 통합 단계별 코드 구현 플랜 작성 지시문 (최종 프롬프트)

아래의 설계 원칙, 코드 규칙, 작성 항목에 따라

각 시스템, UI, 기능별 코드 구현 플랜을 상세하게 작성하시오.

누락/중복/불명확 사항 없이, 팀원 및 AI가 바로 구현 가능하도록 체계적으로 서술할 것.



1. 핵심 설계 원칙 (최상위 규칙)

SOLID 원칙 준수



모든 코드는 확장성, 유지보수성, 모듈성, 재사용성을 최우선으로 설계



단일 책임, 개방-폐쇄, 인터페이스 분리, 의존성 역전 등 SOLID를 반드시 지킨다



UnityEngine 종속성 분리



핵심 도메인 로직(게임플레이 규칙, 수치계산 등)은 UnityEngine 네임스페이스로부터 완전 분리



도메인 로직 내에서 UnityEngine.Debug.Log 대신 추상화된 로깅 인터페이스만 사용



벡터/수학 등은 커스텀 유틸리티 클래스 활용



2. 데이터 및 값 관리 규칙

하드코딩 절대 금지



상수, 파일경로, 리소스명, 사용자표시텍스트 등은 코드에 직접 작성하지 않는다



사용자 텍스트는 반드시 LocalizationManager 등 중앙화 시스템을 통해 처리



그 외 데이터/상수는 ScriptableObject 또는 별도 설정 파일에서 관리



결정적 연산 규칙



스탯/데미지 등 모든 결정적 로직에는 반드시 **스케일된 정수형(int/long)**만 사용



(예: hp_x1000 등 변수명에 스케일 팩터 명시)



오버플로우 위험 항상 확인



float/double은 시각효과, UI 애니메이션 등 비결정적 시각표현에만 제한적 사용



데이터 객체 사용 지침



ScriptableObject는 데이터 공유/설정 목적일 때만 사용 (단순·일회성 값엔 사용 X)



프로덕션 빌드(정식) 런타임 환경에서는 ScriptableObject.CreateInstance 등 동적 생성 금지

(에디터/디버그용 조건부 코드만 허용)



3. 코드 작성 스타일

중복 코드 방지



새로운 기능 구현 전 반드시 기존 코드베이스 분석,

논리적 중복/재작성 없이 리팩터링·확장·재사용 위주로 구현할 것



4. 구현 플랜 항목별 작성법

아래 항목을 표 또는 명확한 구분으로 작성하시오.



① 스크립트 이름

실제 C# 파일명을 명시



② 주요 변수/프로퍼티/이벤트 내역

핵심 public 변수, 프로퍼티, 커맨드, 이벤트 등 인터페이스 표기



③ 파일 및 폴더 위치

프로젝트 내 실제 파일 경로(UXML/USS/SO 등 포함) 명시



④ 상세 구현 방법 및 아키텍처

적용 아키텍처 패턴(MVVM 등)과 View/Model/ViewModel 역할



주요 데이터·이벤트 흐름



외부 시스템 연동 방식(EventChannelSO, SO, Addressable 등)



예외/에러 처리, Pending Question(미정/추가확인 필요점) 포함



⑤ 사용할 네임스페이스

실제 코드에서 사용할 네임스페이스 표기



⑥ 유니티 에디터/셋팅/테스트 관련 체크리스트

프리팹/UXML/SO 등 에디터 연결·생성·배치 필요여부



테스트 씬 또는 Mock 데이터 준비, 단독 테스트 가능여부 등



5. 추가 작성 규칙

더 확인이 필요한 사항, 기획 미정 포인트, 예외/불확정 세부조건은 Pending Question 목록으로 반드시 남길 것



(권장) 각 단계별 리뷰/PR 체크리스트를 문서 하단에 표로 추가할 것



폴더/네임스페이스/이벤트/리소스명 등은 프로젝트 규칙을 반드시 준수
